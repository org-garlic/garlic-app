## Q-T1. Qwen AI의 한국어 성능이 GPT-4나 Claude보다 떨어지지 않나요?

Qwen-Max의 한국어 벤치마크는 GPT-4 대비 약 92~95% 수준입니다. 하지만 저희가 필요한 건 범용 대화가 아니라 **"이 유저에게 어떤 상품을 보여줄 것인가"**라는 좁은 도메인의 판단입니다. 이 영역은 **파인튜닝으로 범용 모델을 앞서는 게 가능**합니다. G마켓 26년 구매 데이터로 파인튜닝한 Qwen은, 범용 GPT-4보다 **우리 유저의 취향을 더 정확히** 맞출 수 있습니다. 비용이 GPT-4 대비 1/5 수준이라 MAU 1000만 규모에서 실질적으로 운영 가능한 유일한 선택지입니다.

## Q-T2. 실시간 UI 재조합의 응답 속도가 사용자 경험을 해치지 않나요?

**p95 레이턴시 목표는 500ms 이내**입니다. **3단계 캐시 전략**:
- **L1 (Edge)**: 유저 DNA 프로필 + 레이아웃 JSON을 Redis에 캐시 → 50ms
- **L2 (배치)**: 매일 새벽 PAI에서 전체 유저 DNA 갱신
- **L3 (실시간)**: 세션 중 행동 변화만 Flink로 델타 보정

앱 진입 시 L1 캐시 히트율 95%+를 목표로 합니다. Qwen 추론은 **스크롤 2페이지 이후의 추천 상품**에만 사용되므로 초기 로딩에는 영향 없습니다.

## Q-T3. 26년 데이터라고 하는데, 실제로 AI 학습에 쓸 수 있는 품질인가요?

솔직히 전부 쓸 수 있는 건 아닙니다. 실질적으로 학습에 활용하는 건 **최근 5년 데이터 + 10년간 누적 구매 패턴 요약**입니다. 하지만 핵심은 **"같은 유저의 10년치 구매 이력"**이 있다는 점입니다. 쿠팡은 3~4년치만 있고, 네이버는 결제 데이터만 있습니다. **유저의 취향 변화 곡선**을 10년 단위로 볼 수 있는 건 G마켓뿐입니다.

## Q-T4. On-Premise AI 구축은 클라우드 대비 비용이 훨씬 많이 드는 거 아닌가요?

**MAU 1000만 × 세션당 2회 추론** 규모에서는 **18개월 시점에 클라우드 API 비용을 역전**합니다.
- 클라우드 API: MAU 1000만 기준 연 **120억+** (GPT-4 기준)
- On-Premise Qwen: 초기 인프라 **40억** + 연 운영 **15억**

3년 TCO로 보면 On-Premise가 **60% 저렴**합니다. **유저 구매 데이터가 외부 API로 나가지 않는다**는 점은 법적 리스크 면에서도 결정적 이점입니다.

## Q-T5. 갈릭 DNA 6축 프로파일링의 정확도는 어떻게 검증하나요?

**오프라인 A/B 테스트 + 온라인 MAB**으로 이중 검증합니다.
- MVP: 기존 추천 vs DNA 추천 **2그룹 A/B** → 클릭률/전환율 비교
- 운영: **Thompson Sampling** 기반 MAB로 자동 최적화
- 정확도 지표: **추천 CTR 8%+**, **추천 CVR 15%+**

기존 G마켓 협업필터링 추천 CTR 약 3.2%이므로, DNA 추천이 **2.5배 이상**이면 MVP 성공으로 판단합니다.

## Q-T6. 벡터 DB가 상품 100만 개 규모에서 실시간 검색이 가능한가요?

AnalyticDB는 **10억 벡터** 규모에서도 **10ms 이내** 검색이 검증된 서비스입니다. G마켓 활성 상품 **200~300만 개** 기준 여유 충분합니다.
- 상품 벡터: Qwen-VL 임베딩 (768차원)
- **Cosine Similarity TOP-K**: 상품 300만 기준 **p99 < 15ms**

알리바바 타오바오가 **10억 상품**에서 실제 운영 중인 동일 스택입니다.

## Q-T7. AI 추천이 편향되면 셀러 공정성 문제가 생기지 않나요?

- **Exploration 20%**: DNA 무관하게 **신규 셀러/신상품 노출**
- **셀러 대시보드**: DNA 매칭 분포 확인 + 상품 최적화 도구
- **공정성 감사 로그**: 월간 **카테고리별 노출 편중도** 모니터링

"AI가 광고비 대신 취향으로 노출을 결정"하는 것이 **셀러에게도 더 공정한 구조**입니다.

## Q-T8. 개인정보보호법 강화 추세에서 이 수준의 행동 데이터 수집이 가능한가요?

**3단계 데이터 분류 체계**:
- **Level 1 (동의 불요)**: 비식별 세션 데이터 (클릭/스크롤/체류)
- **Level 2 (마케팅 동의)**: 구매 이력 기반 추천
- **Level 3 (별도 동의)**: DNA 프로파일 생성 → **명시적 옵트인 + 삭제권**

On-Premise 구축으로 **데이터가 해외로 나가지 않는다**는 점도 규제 대응에 유리합니다.

## Q-T10. AI 추천이 실패하면 복구 플랜은?

**3중 안전장치**:

- **킬 스위치**: 원클릭 AI 비율 0~100% 조절 Feature Flag → 30초 내 롤백

## Q-T11. Qwen 파인튜닝에 필요한 학습 데이터 형태는?

- **구매 시퀀스**: 유저별 2년 구매 이력 시간순. **8,000만 시퀀스**
- **클릭-비클릭 쌍**: 같은 검색어 클릭 vs 스킵. **3억 쌍**
- **리뷰 감성**: 별점 + 텍스트 감성 태깅. **5,000만 건**

**Instruction Tuning + LoRA 파인튜닝**: 4~8 GPU 기준 약 **72시간** 1차 학습 완료.

## Q-T12. 실시간 스트리밍(Flink)이 장애나면 전체 추천이 멈추나요?

**Graceful Degradation** 설계:
- Flink 장애 → Redis 캐시 DNA로 추천 유지 (최대 6시간 지연)
- Redis도 장애 → 일간 DNA 스냅샷 폴백
- 전부 장애 → 규칙 기반 추천 자동 전환

**단일 장애점(SPOF)이 없습니다.**

- **SSR**: 크롤러에게 고정 레이아웃 + JSON-LD 서빙

컴포넌트 풀 자체에 **WCAG 2.1 AA** 내장. 어떤 variant든 시맨틱 HTML + ARIA 보장.

## Q-T14. 기기 변경/로그아웃 시 DNA가 리셋되나요?

**3단계 ID 통합**:
- 로그인: 회원 ID에 DNA 영구 귀속
- 비로그인: 디바이스 핑거프린트 + 쿠키 임시 DNA → 로그인 시 자동 병합
- 크로스 디바이스: DNA는 유저 단위, 기기는 컨텍스트 변수

## Q-T15. Qwen-VL 이미지 분석의 정확도와 비용은?

- **정확도**: 패션 Top-3 스타일 태그 **89%** (알리바바 벤치)
- **비용**: 이미지 1장 **0.3원**, 일 1만 개 기준 월 **90만 원**
- 상품 등록 시 **1회 배치 처리** (실시간 아님)

텍스트로 잡을 수 없는 **비주얼 취향 매칭**이 핵심 차별점.

## Q-T16. A/B 테스트 자동화에서 잘못된 UI 노출 리스크는?

**3단계 안전 롤아웃**:
- **Shadow**: 병렬 로깅만 (실서빙 X)
- **Canary**: 1%에게만 노출 → 이상 시 자동 롤백
- **Ramp-up**: 1→5→20→50→100% (KPI 95% 미만 시 자동 중단)

90%는 항상 **검증된 최적 UI**를 봅니다.

## Q-T20. 이 기술 스택이 작동하는 걸 어떻게 증명하나요?
